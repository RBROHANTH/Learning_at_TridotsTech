Essential Python Code Snippets

Congrats on your internship at Tridots! Since you are handling 10 different CRUD (Create, Read, Update, Delete) and state-tracking assignments, you don't need 10 completely different codebases. They all boil down to a few core Python mechanics: dictionaries for fast lookups, lists of objects, input validation, and conditional state updates.

Instead of overwhelming you, here are the essential, structural Python code snippets you must know to build all 10 systems cleanly.

1. The Core Data Structure: Nested Dictionaries

Essential for: Student, Employee, Contact, Inventory Systems

For systems like the Student, Employee, Contact, and Inventory systems, a nested dictionary is the cleanest way to store data using a unique ID (like a roll number, employee ID, or barcode) as the key.

# Database initialization
database = {}

# 1. Add / Update (Create/Update)
# If the ID exists, it updates; if not, it creates a new entry.
database["STU101"] = {"name": "Alice", "marks": 85, "grade": "A"}

# 2. Search / Read
student_id = "STU101"
if student_id in database:
    print(f"Found: {database[student_id]['name']}")
else:
    print("Record not found.")

# 3. Delete
# Using .pop() with a default value prevents KeyError crashes if the ID doesn't exist
removed = database.pop("STU101", None)

2. State & Balance Modifications (Atomic Updates)

Essential for: ATM, Expense Tracker, Inventory (Stock)

For the ATM, Expense Tracker, and Inventory (Stock) systems, you need to safely adjust numeric values while validating limits (e.g., preventing a negative balance or selling more stock than you have).

balance = 1000.0   # Or stock = 50
withdraw_amount = 400.0

# Always check the condition BEFORE modifying the state
if withdraw_amount <= 0:
    print("Invalid amount.")
elif withdraw_amount > balance:
    print("Insufficient funds.")
else:
    balance -= withdraw_amount
    print(f"Success! New balance: {balance}")

3. Modifying Lists Inside Objects

Essential for: Library, Railway, Hotel Booking

For the Library, Railway, and Hotel systems, you are tracking the state of an asset (Book, Seat, Room) that toggles between available and unavailable.

# A dictionary representing rooms or seats
rooms = {
    "101": {"type": "Deluxe", "is_booked": False},
    "102": {"type": "Suite", "is_booked": True}
}

room_to_book = "101"

# Check-in / Booking logic
if room_to_book in rooms:
    if not rooms[room_to_book]["is_booked"]:
        rooms[room_to_book]["is_booked"] = True
        print(f"Room {room_to_book} successfully booked!")
    else:
        print("Sorry, room is already occupied.")

4. Categorized Filtering & Aggregation

Essential for: Expense Tracker, Quiz Application

For the Expense Tracker, you need to filter data based on a category and sum up totals.

expenses = [
    {"amount": 50, "category": "Food"},
    {"amount": 20, "category": "Transport"},
    {"amount": 100, "category": "Food"},
]

# 1. Calculate Total Expense
total = sum(item["amount"] for item in expenses)

# 2. Search/Filter by Category
search_category = "Food"
filtered_expenses = [item for item in expenses if item["category"] == search_category]

5. Simple Iteration Mapping

Essential for: Quiz Application

For the Quiz Application, you need to pair a loop with data tuple/dictionary unpacking to evaluate user input dynamically.

quiz_bank = [
    {"q": "What is 2+2?", "a": "4"},
    {"q": "What language is this?", "a": "python"}
]

score = 0
for item in quiz_bank:
    print(item["q"])
    user_ans = input("Your answer: ").strip().lower()
    
    if user_ans == item["a"].lower():
        score += 1

💡 Intern Pro-Tip for Tridots Assignments:

Instead of writing endless if/elif blocks for your command-line menus, look into Python's Match-Case (introduced in Python 3.10). It will make your internship code look incredibly clean and professional: 

choice = input("Enter choice (1-4): ")
match choice:
    case "1": check_balance()
    case "2": deposit_money()
    case "3": withdraw_money()
    case "4": print("Exiting...")
    case _: print("Invalid option chosen.") # Fallback wildcard
